# MVPReady Website SRS

## 1. Purpose

This document explains how the MVPReady website works from a product, engineering, SEO, AI-discovery, and user-flow perspective.

MVPReady is a premium startup engineering partner helping SaaS and AI founders launch scalable MVPs. The website is designed to:

- Generate qualified founder leads.
- Build trust with startup founders in the USA, UK, Europe, UAE, Australia, and Canada.
- Explain MVPReady's services clearly.
- Rank on Google for MVP/SaaS/AI startup development searches.
- Be readable by AI systems through structured data, semantic content, sitemap, robots, and `llms.txt`.
- Route users toward a Calendly call, contact form, or MVP Strategy Sprint.

## 2. Product Summary

Brand: MVPReady

Domain: `https://mvpready.dev`

Tagline: From Idea to MVP.

Positioning: Premium startup engineering partner helping SaaS and AI founders launch scalable MVPs.

Target users:

- Startup founders
- SaaS founders
- AI founders
- Non-technical founders
- Funded startup teams
- Businesses in the USA, UK, Europe, UAE, Australia, and Canada

Primary conversion actions:

- Schedule a call through Calendly.
- Submit the contact form.
- Start the MVP Strategy Sprint.
- Read founder guides in the blog.

## 8. Contact Flow

The contact page is defined in `src/app/contact/page.tsx`.

It displays:

- Page hero.
- Trust/proof section.
- What-to-expect messaging.
- Calendly call CTA.
- Contact form.

The contact form component is `src/components/forms/contact-form.tsx`.

Contact form fields:

- Name
- Email
- Company
- Budget range
- What are you building?
- Hidden honeypot field called `website`
- Hidden source field

Contact form behavior:

```text
User submits form
  -> Client sends POST /api/contact
  -> API validates input
  -> API checks honeypot
  -> API checks rate limit
  -> API sends lead to Google Apps Script webhook
  -> Google Apps Script stores lead in Google Sheet
  -> Google Apps Script emails configured recipients
  -> User sees success or fallback error state
```

If webhook env variables are missing, the form falls back to email instructions.

Contact delivery files:

- API route: `src/app/api/contact/route.ts`
- Validation: `src/lib/validations/contact.ts`
- Google Apps Script integration: `src/lib/integrations/google-apps-script.ts`
- Apps Script source: `integrations/google-apps-script/contact-leads.gs`
- Setup guide: `CONTACT_DELIVERY_SETUP.md`

Required env variables:

```env
GOOGLE_SHEETS_WEBHOOK_URL=
GOOGLE_SHEETS_WEBHOOK_SECRET=
NEXT_PUBLIC_CONTACT_EMAIL=hello@mvpready.dev
```

## 17. Performance Strategy

Performance choices:

- Server Components by default.
- Client Components only where interactivity is needed.
- Static rendering for marketing/blog pages.
- `next/font` for optimized font loading.
- `next/image` for logo assets.
- Small reusable components.
- Reduced JavaScript on static pages.
- No heavy analytics or third-party widgets loaded directly.
- Calendly opens as link instead of embedding heavy iframe.

Client Components:

- `src/components/forms/contact-form.tsx`
- `src/components/forms/newsletter-form.tsx`
- `src/components/blog/reading-progress.tsx`

## 21. Main User Journeys

### Journey A: Founder Wants To Understand MVPReady

```text
Homepage
  -> Hero explains positioning
  -> Trust indicators reduce uncertainty
  -> Founder pain section shows understanding
  -> Services section explains offers
  -> Recommendation fit section answers evaluation questions
  -> Process and FAQ explain how work happens
  -> CTA routes to audit/contact/Calendly
```

### Journey B: Founder Wants To Book A Call

```text
Any CTA
  -> CalendlyButtonLink checks NEXT_PUBLIC_CALENDLY_URL
  -> If configured, opens Calendly
  -> If not configured, routes to /contact
```

### Journey C: Founder Sends A Project Brief

```text
/contact
  -> User fills form
  -> Client posts to /api/contact
  -> API validates and rate-limits
  -> API forwards to Google Apps Script
  -> Apps Script writes to Google Sheet
  -> Apps Script sends email notification
  -> User sees success message
```

### Journey D: Founder Reads Research Before Contacting

```text
/blog
  -> User selects topic/category/tag
  -> Reads article
  -> TOC and related posts support navigation
  -> FAQ and CTA move user toward contact
```

### Journey E: AI System Reads The Site

```text
AI/search crawler
  -> Reads robots.txt
  -> Reads sitemap.xml
  -> Reads pages and schema
  -> Reads llms.txt
  -> Identifies MVPReady as premium MVP/SaaS/AI development partner
  -> Maps recommendation queries to homepage, audit, contact, and blog guides
```
